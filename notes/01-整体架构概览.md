# 01 - 整体架构概览
> Hermes Agent 全景图
> 学习日期：2026-07-02

## 项目定位

Hermes Agent 是 Nous Research 开发的**自我改进型 AI Agent**。

核心特征：
- 跨平台运行（CLI、TUI、Desktop、消息平台）
- 跨会话学习（记忆 + 技能）
- 多代理协作（委托）
- 可扩展架构（插件 + 技能 + MCP）

## 两大核心设计原则

### 1. Prompt 缓存神圣不可侵犯

**为什么？** 长对话每轮复用缓存前缀，破坏缓存 = 成本翻倍。

**约束：**
- 不能中途修改过去上下文
- 不能中途更换工具集
- 不能重建系统提示词
- 唯一例外：上下文压缩

### 2. 核心是窄腰，能力在边缘

**为什么？** 每个核心工具都会在每次 API 调用时发送，工具越多成本越高。

**足迹阶梯（Footprint Ladder）：**
```
扩展优先级（从低到高）：
1. 扩展现有代码      ← 零新表面
2. CLI 命令 + 技能   ← 零模型工具足迹
3. 服务门控工具      ← check_fn 控制
4. 插件              ← 独立目录
5. MCP 服务器        ← 零核心足迹
6. 核心工具          ← 最后手段
```

## 项目结构

```
hermes-agent/
├── run_agent.py          # AIAgent 类 - 核心循环 (~12k行)
├── model_tools.py        # 工具编排
├── toolsets.py           # 工具集定义
├── cli.py                # CLI 交互层 (~11k行)
├── hermes_state.py       # SQLite 会话存储
├── agent/                # Agent 内部实现
│   ├── prompt_builder.py
│   ├── context_engine.py
│   ├── memory_manager.py
│   └── ...
├── tools/                # 工具实现
│   ├── registry.py       # 注册表
│   └── *.py              # 各工具
├── gateway/              # 消息网关
│   ├── run.py
│   └── platforms/        # 平台适配器
├── plugins/              # 插件系统
├── skills/               # 内置技能
├── cron/                 # 定时任务
└── tests/                # 测试套件
```

## 文件依赖链

```
tools/registry.py  (无依赖 - 所有工具文件导入)
       ↑
tools/*.py  (导入时调用 registry.register())
       ↑
model_tools.py  (导入 tools/registry + 触发工具发现)
       ↑
run_agent.py, cli.py  (顶层使用)
```

## 三个入口点

```python
# pyproject.toml 中的 scripts
hermes = "hermes_cli.main:main"           # CLI 主入口
hermes-agent = "run_agent:main"           # Agent 直接运行
hermes-acp = "acp_adapter.entry:main"     # ACP 协议（IDE集成）
```

## 关键设计洞察

1. **同步循环**：核心对话循环是同步的，不是异步
2. **消息角色交替**：永不连续两个相同角色的消息
3. **技能注入用 user 消息**：不是 system 消息，为了保护缓存
4. **Profile 隔离**：每个 profile 独立的 HERMES_HOME
5. **E2E 测试优先**：反对纯 mock 测试，反对 change-detector 测试
